# 確定申告プレファイリングチェック

e-Tax入力を開始する前に、申告内容の漏れがないかチェックリストで確認する。
チェックリストは国税庁QA（https://www.nta.go.jp/taxes/shiraberu/shinkoku/qa/13.htm）
および関連タックスアンサーに基づく。

## 対象年度

年度ラベル: `$ARGUMENTS` （例: `2025_R7`）

## 処理手順

### 1. データ読込み

以下のファイルを読み込み、現在の申告データの状況を把握する：

- `years/{year_label}/data/match_results.json` — 突合結果
- `years/{year_label}/data/insurance_deductions.json` — 社会保険料データ
- `years/{year_label}/data/reimbursements.json` — 補填金額
- `years/{year_label}/output/tax_summary.md` — 現在の申告サマリー（あれば）
- `years/{year_label}/output/etax_entry_guide.md` — 入力ガイド（あれば）

### 2. チェックリスト提示

以下のチェックリストを**インタラクティブに**ユーザーに提示する。
各カテゴリについて、データから自動判定できるものは判定済みとし、
人間の確認が必要なものは質問する。

---

## チェックリスト

### A. 確定申告の必要性（NTA QA参照）

- [ ] **給与所得者の確定申告要否**: 年収2,000万円超、2ヶ所以上の給与、給与外所得20万円超のいずれかに該当するか
- [ ] **退職所得の有無**: 年内に退職し年末調整を受けていない場合は申告が必要
- [ ] **年末調整の実施確認**: 源泉徴収票を確認し、年末調整が完了しているか

### B. 所得の確認（NTA タックスアンサー参照）

- [ ] **給与所得**: 源泉徴収票の金額を確認（No.1400）
- [ ] **雑所得の有無**: 副業収入、原稿料、暗号資産取引等（No.1500, No.1906, No.2010）
- [ ] **一時所得の有無**: 生命保険の満期金、懸賞金等（No.1490）
- [ ] **配当所得の有無**: 株式配当等の課税方式選択（No.1120）
- [ ] **譲渡所得の有無**: 不動産売却、株式売却等（No.1440, No.1600）

### C. 所得控除の該当確認

#### C-1. 医療費控除
- [ ] **年間医療費の合計**: 10万円（または所得の5%）を超えているか
- [ ] **領収書の整理**: 全ての領収書がスキャン・JSON化されているか
- [ ] **マイナポータル連携**: 医療費通知データを取得済みか
- [ ] **補填金額の確認**: 保険金・自治体補助金等を差し引いているか
- [ ] **セルフメディケーション税制との選択**: どちらが有利か検討したか

#### C-2. 社会保険料控除
- [ ] **国民年金保険料**: 控除証明書（日本年金機構ハガキ）があるか
- [ ] **国民健康保険税**: 納付額証明書があるか
- [ ] **共済組合掛金**: 源泉徴収票の「社会保険料等の金額」に含まれているか確認
- [ ] **家族分の保険料**: 生計を一にする家族の分も申告可能

#### C-3. 生命保険料控除
- [ ] **控除証明書の有無**: 各保険会社から届いた証明書があるか
- [ ] **年末調整での処理確認**: 源泉徴収票の「生命保険料の控除額」欄を確認
- [ ] **新旧制度の区分**: 証明書に記載の新旧区分を確認
- [ ] **証明額 vs 申告額**: 12月末時点の見込額（申告額）を使用しているか

#### C-4. 地震保険料控除
- [ ] **控除証明書の有無**: 保険会社からの証明書があるか
- [ ] **年末調整での処理確認**: 源泉徴収票の「地震保険料の控除額」欄を確認
- [ ] **火災保険料との区別**: 地震保険料のみが控除対象（火災保険は対象外）

#### C-5. 寄附金控除（ふるさと納税等）
- [ ] **寄附金受領証明書/XML**: ふるさと納税サイトからダウンロード済みか
- [ ] **ワンストップ特例の無効化**: 確定申告する場合、ワンストップ特例は無効になる
- [ ] **寄附先の確認**: 全ての寄附先が含まれているか

#### C-6. 配偶者控除・扶養控除
- [ ] **配偶者の所得確認**: 配偶者控除/特別控除の適用要件（No.1191, No.1195）
- [ ] **扶養親族の確認**: 16歳以上の扶養家族の有無（No.1145, No.1177）
- [ ] **障害者控除の確認**: 該当者がいるか（No.1170）

#### C-7. 住宅ローン控除
- [ ] **住宅借入金等特別控除**: 初年度は確定申告が必要（No.1211-1, No.2040）
- [ ] **2年目以降**: 年末調整で処理済みか確認

#### C-8. その他の控除
- [ ] **基礎控除**: 所得2,400万円以下なら48万円（No.1199）
- [ ] **ひとり親控除**: 該当するか（No.1171）
- [ ] **災害減免**: 災害による損失がある場合

### D. 必要書類の準備状況

- [ ] **源泉徴収票**: 勤務先から受領済みか
- [ ] **マイナンバーカード**: e-Tax利用に必要
- [ ] **各種控除証明書**: 上記C項で必要な証明書がすべて揃っているか
- [ ] **振込先口座情報**: 還付金の振込先

### E. 復興特別所得税

- [ ] **復興特別所得税の認識**: 所得税額×2.1%が加算される

---

### 3. 自動判定ロジック

データから自動的に確認できる項目：

| チェック項目 | 確認方法 |
|-------------|---------|
| 医療費合計が10万円超か | `match_results.json` の合計額を計算 |
| 補填金額が反映されているか | `reimbursements.json` の存在と内容確認 |
| マイナポータルデータがあるか | `mynaportal.json` の存在確認 |
| 社会保険料データがあるか | `insurance_deductions.json` の存在確認 |
| ふるさと納税XMLがあるか | `sources/other/` の `.xml` ファイル確認 |
| 全領収書がJSON化されているか | `sources/receipts/` と `data/batches/` の件数比較 |

### 4. 出力

チェック結果を `years/{year_label}/output/prefiling_checklist.md` に出力する。

出力内容：
- 各項目の確認結果（自動判定結果 + ユーザー回答）
- 未確認・要対応項目のハイライト
- 次のアクション一覧

### 5. インタラクション

ユーザーに以下の質問を**順番に**行う（自動判定できないもの）：

1. 「今年、給与以外の所得（副業、暗号資産、一時所得等）はありましたか？」
2. 「年末調整で処理済みの控除を教えてください（源泉徴収票で確認）」
3. 「住宅ローン控除の申告は必要ですか？」
4. 「上記チェックリストで追加すべき項目はありますか？」
